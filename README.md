# Легенда

Разрабатывается последний писк Инфраструктуры Экосистемы, младший брат виджета галочки согласия, встречайте: **виджет перевода текста в upper case**.

Мы решили, что двухступенчатая схема `importWidgetModule + render` тут не нужна и упростили ее до одноступенчатой, чтобы упростить интеграцию в виджет WOK. Строка передается прям в `importWidgetModule`, в `dependencies`, и результат возвращается в объекте модуля. Вся полезная работа делается прям в `createWidgetApi`.

Код виджета пока в разработке. Но для тестирования интеграции мы подготовили его эмуляцию, с нужной сигнатурой. См. `./src/helpers.ts`.

```tsx
const importWidgetModule: (text: string) => Promise<{
    value: string,
    dispose: () => Promise<void>;
}>
```

Как и в любом виджете, importWidgetModule не просто делает свое дело, а:

- **ждет** сетевых запросов. Мы это эмулируем через задержку 200 мс
- **жрет** память и айфреймы при загрузке. Мы это эмулируем через счетчик "Side effect count". Считай, это количество айфреймов на странице

Чтобы упростить интеграцию в виджет WOK, `OperationResult` решили не использовать: ошибок от виджета в МVP не планируется.

И да, токены пока не завезли: это старая версия платформы, с асинхронной функцией `dispose`. Но, как по мне, без них тут будет даже понятнее. Кстати, тут функция `dispose` идемпотентная: можно диспоузить много раз, за счет функции `memo`. Это чтоб было больше похоже на поведение `tcs.cancel()`.

# Задача 0

Форкни этот проект и добавь очистку.

Правила:

- очистка должна срабатывать, когда меняется зависимость у атома -- текст в input-е
- очистка должна срабатывать по снятию галочки
- очистка должна быть не отдельным эффектом или вызовом из реакта, а как-то триггериться в jotai
- нельзя добавлять в сеттер атомов с текстом и галочкой никакую логику: они должны оставаться PrimitiveAtom-ами и не должны знать, что влияют на отмену и пересоздание, как не знают и сейчас
- нельзя использовать глобальные мутабельные штуки: если что-то хранишь, храни в store
- когда нужно поменять значение, сначала надо дождаться конца очистки старого, а только потом запускать новый `importWidgetModule`

Пока забей на гоночные сценарии, когда что-нибудь быстро меняется, про это будут отдельные шаги. Пока печатай в input медленно, по 1 буковке.
